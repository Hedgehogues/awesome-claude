# PROCESS.md — процессная модель

# Контекст
Документ описывает требования к процессной модели приложения. Цель — статлес поведение и предсказуемость запуска и остановки.

## Правила
- Процессы не хранят критичное состояние локально.
- Любое состояние хранится во внешних сервисах.
- Приложение должно корректно завершаться по сигналам.
- Миграции и одноразовые задачи выполняются отдельными командами.
- Локальные файлы допускаются только как временные артефакты.
- Конфигурация и окружение читаются при старте процесса.

## Антипаттерны (запрещено)
- Долгоживущие файлы состояния внутри контейнера.
- Встроенные cron-задачи без контроля процесса.
