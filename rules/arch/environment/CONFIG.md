# CONFIG.md — конфигурация

# Контекст
Документ описывает правила управления конфигурацией и секретами. Цель — обеспечить корректное разделение окружений и безопасное хранение параметров.

## Правила
- Конфигурация задается через переменные окружения или явные файлы конфигурации.
- Секреты хранятся вне репозитория.
- Значения по умолчанию допускаются только для некритичных параметров.
- Конфигурация валидируется при старте приложения.
- Переменные окружения документируются и имеют явное назначение.
- Разные окружения используют одинаковые ключи с разными значениями.

## Антипаттерны (запрещено)
- Секреты в коде или в истории репозитория.
- Неполные или невалидируемые конфиги.
