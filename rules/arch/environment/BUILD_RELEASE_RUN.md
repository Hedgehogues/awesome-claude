# BUILD_RELEASE_RUN.md — сборка, релиз, запуск

# Контекст
Документ разделяет три стадии доставки кода: сборка (build), релиз (release), запуск (run). Цель — воспроизводимость и откатываемость каждого деплоя.

## Правила
- Сборка, релиз и запуск — строго разделённые стадии.
- Сборка: код + зависимости → артефакт (Docker-образ).
- Релиз: артефакт + конфигурация окружения → готовый к запуску пакет.
- Запуск: запуск конкретного релиза в окружении.
- Каждый релиз имеет уникальный идентификатор (тег образа, SHA коммита).
- Релиз неизменяем — любое изменение создаёт новый релиз.
- Откат выполняется переключением на предыдущий релиз, а не патчем текущего.
- Артефакт сборки не зависит от целевого окружения.

## Антипаттерны (запрещено)
- Изменение кода или зависимостей на стадии запуска.
- Сборка, привязанная к конкретному окружению.
- Отсутствие версионирования релизов.
- Ручные правки внутри запущенного контейнера.
