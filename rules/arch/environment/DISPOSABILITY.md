# DISPOSABILITY.md — одноразовость процессов

# Контекст
Документ описывает требования к быстрому старту и корректному завершению процессов. Цель — устойчивость к перезапускам, деплоям и масштабированию.

## Правила
- Процессы запускаются быстро (секунды, не минуты).
- Процессы корректно завершаются по SIGTERM.
- При получении SIGTERM:
  - web-процесс прекращает приём новых запросов и дорабатывает текущие.
  - worker-процесс возвращает задачу в очередь.
- Процессы устойчивы к внезапному завершению (SIGKILL, OOM, сбой хоста).
- Незавершённые операции не оставляют систему в неконсистентном состоянии.
- Идемпотентность операций обеспечивает безопасность повторного выполнения.
- Graceful shutdown timeout задаётся явно в конфигурации.

## Антипаттерны (запрещено)
- Долгая инициализация при старте (тяжёлые прогревы, синхронные миграции).
- Игнорирование SIGTERM.
- Потеря данных при перезапуске процесса.
- Длительные операции без механизма возобновления.
