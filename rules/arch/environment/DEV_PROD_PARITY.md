# DEV_PROD_PARITY.md — паритет окружений

# Контекст
Документ фиксирует требования к минимальному расхождению между dev, test и prod окружениями. Цель — ранее обнаружение проблем и предсказуемое поведение при деплое.

## Правила
- Dev-окружение максимально приближено к prod.
- Одни и те же backing services: PostgreSQL, S3-совместимое хранилище (MinIO локально, AWS S3 в prod).
- Одинаковые версии сервисов (PostgreSQL 16 везде).
- Одинаковый рантайм: Python 3.12, одинаковый базовый образ.
- Код деплоится одним и тем же механизмом (Docker-образ) во всех окружениях.
- Различия между окружениями выражаются только в конфигурации.
- Тестовое окружение использует реальные сервисы (docker-compose.test.yml), а не in-memory заменители.

## Три разрыва, которые минимизируем
- Временной: код деплоится быстро после написания.
- Кадровый: разработчик участвует в деплое.
- Инструментальный: одинаковые инструменты в dev и prod.

## Антипаттерны (запрещено)
- SQLite в тестах при PostgreSQL в prod.
- Файловая система вместо S3 в dev.
- Существенно отличающиеся версии backing services между окружениями.
- Тесты только на моках без интеграционной проверки с реальными сервисами.
