# DEPENDENCIES.md — зависимости

# Контекст
Документ фиксирует правила управления зависимостями. Он нужен, чтобы сборка была воспроизводимой, а окружение — стабильным и предсказуемым.

## Правила
- Все зависимости описываются в манифесте проекта.
- Версии фиксируются и воспроизводимы (lockfile).
- Системные зависимости также документируются.
- Обновление зависимостей происходит контролируемо.
- Разделяются dev и prod зависимости.
- Внешние бинарники устанавливаются через явный механизм.

## Антипаттерны (запрещено)
- Зависимости, установленные вручную без фиксации.
- Использование разных версий библиотек в разных окружениях.
