# Commands

## Контекст
Команда — это вход в write-слой, описывающий намерение изменить состояние
домена. Это не факт и не событие, а запрос на действие в настоящем времени:
CreateUser, PublishAgent, PayInvoice. Контекст определяет, что команда должна
содержать только данные, проходить валидацию и применяться к одному агрегату,
после чего либо приводит к изменениям, либо отклоняется. Раздел задает место
команд в цепочке исполнения и отделяет их от чтения и событий, чтобы write-логика
оставалась предсказуемой, идемпотентной и трассируемой.

## Правила
- Команда может быть отклонена. Валидация и бизнес-правила — норма, а не
  исключение.
- Команда адресована одному агрегату. Один command → один Aggregate Root.
  Массовые команды — архитектурная ошибка.
- Команда не содержит бизнес-логики. Это DTO с данными, не «умный объект».
- Команда валидируется до исполнения.
- Техническая валидация: формат, обязательные поля.
- Доменная валидация — внутри агрегата.
- Команда исполняется синхронно. Приняли → попытались применить → либо успех,
  либо отказ. Асинхронность — на уровне эффектов.
- Команда приводит к изменению состояния агрегата и/или генерации событий.
  Команда без эффекта — мусор.
- Команда не возвращает данные агрегата. Максимум — OK / ERROR / aggregate_id /
  version. Чтение — отдельным запросом.
- Команда идемпотентна или имеет idempotency key. Повторная отправка не должна
  ломать состояние.
- Команда обрабатывается через Command Handler / Application Service.
  Контроллеры и очереди туда не лезут.
- Команды не ходят напрямую в БД. Только через агрегаты и репозитории.
- Команды не вызывают другие команды напрямую. Оркестрация — через саги или
  процесс-менеджеры.
- Ошибки команд — доменные, явные, типизированные. Не
  Exception("что-то пошло не так").
- Команды логируются и трассируются. command_id, correlation_id, actor_id
  обязательны.
- Команды не публикуются как события. Команда — вход, событие — выход.
  Путать нельзя.

## Обработка команд
- Command Handler загружает агрегат через репозиторий.
- Command Handler вызывает метод агрегата.
- Command Handler сохраняет агрегат.
- Command Handler не содержит бизнес-решений.

## Связь с событиями
- Команда может породить 0..N доменных событий.
- Событие никогда не превращается обратно в команду автоматически.
- SAGA не приветствуется.
