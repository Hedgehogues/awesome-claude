# Components

## Контекст
Раздел описывает ключевые архитектурные компоненты доменной модели и правила
их взаимодействия. Эти материалы задают границы ответственности, способы чтения
и записи, а также требования к консистентности и интеграциям.

## Содержимое
- `AGREGATES.md` — правила для агрегатов и границ транзакций.
- `AGGREGATE_STRUCTURE.md` — полная структура агрегата: entity, repo, models, тесты, чеклист.
- `COMMANDS.md` — требования к командам и обработчикам.
- `DOMAIN.md` — принципы описания доменной модели.
- `EVENTS.md` — правила для доменных событий и публикации.
- `QURIES.md` — правила для read-слоя и запросов.
