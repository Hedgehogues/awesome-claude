# Generalizations

## Контекст
Обобщение связано с управлением иерархиями наследования и общими контрактами.
Контекст этой группы помогает понять, когда стоит поднимать поведение вверх по
иерархии, когда — опускать его вниз, а когда — выделять интерфейсы или заменять
наследование делегированием. Эти рефакторинги важны, потому что неправильная
структура иерархий быстро приводит к дублированию, жесткой связанности и
неустойчивому дизайну. Блок задает безопасные способы эволюции общего поведения,
сохраняя читаемость и расширяемость модели.

## Рефакторинги
### Pull Up Field
**Проблема:** Два класса имеют одно и то же поле.
**Решение:** Переместите поле в суперкласс, убрав его из подклассов.

### Pull Up Method
**Проблема:** Подклассы имеют методы, которые делают схожую работу.
**Решение:** В этом случае нужно сделать методы идентичными, а затем переместить
их в суперкласс.

### Pull Up Constructor Body
**Проблема:** Подклассы имеют конструкторы с преимущественно одинаковым кодом.
**Решение:** Создайте конструктор в суперклассе и вынесите в него общий
для подклассов код. Вызывайте конструктор суперкласса в конструкторах подкласса.

### Push Down Method
**Проблема:** Поведение, реализованное в суперклассе, используется только одним
или несколькими подклассами.
**Решение:** Переместите это поведение в подклассы.

### Push Down Field
**Проблема:** Поле используется только в некоторых подклассах.
**Решение:** Переместите поле в эти подклассы.

### Extract Subclass
**Проблема:** Класс имеет фичи, которые используются только в определённых случаях.
**Решение:** Создайте подкласс и используйте его в этих случаях.

### Extract Superclass
**Проблема:** У вас есть два класса с общими полями и методами.
**Решение:** Создайте для них общий суперкласс и перенесите туда одинаковые
поля и методы.

### Extract Interface
**Проблема:** Несколько клиентов пользуются одной и той же частью интерфейса
класса. Либо в двух классах часть интерфейса оказалась общей.
**Решение:** Выделите эту общую часть в свой собственный интерфейс.

### Collapse Hierarchy
**Проблема:** У вас есть некая иерархия классов, в которой подкласс мало чем
отличается от суперкласса.
**Решение:** Слейте подкласс и суперкласс воедино.

### Form Template Method
**Проблема:** В подклассах реализованы алгоритмы, содержащие похожие шаги
и одинаковый порядок выполнения этих шагов.
**Решение:** Вынесите структуру алгоритма и одинаковые шаги в суперкласс,
а в подклассах оставьте реализацию отличающихся шагов.

### Replace Inheritance with Delegation
**Проблема:** У вас есть подкласс, который использует только часть методов
суперкласса или не хочет наследовать его данные.
**Решение:** Создайте поле и поместите в него объект суперкласса, делегируйте
выполнение методов объекту-суперклассу, уберите наследование.

### Replace Delegation with Inheritance
**Проблема:** Класс содержит множество простых делегирующих методов ко всем
методам другого класса.
**Решение:** Сделайте класс наследником делегата, после чего делегирующие методы
потеряют смысл.
