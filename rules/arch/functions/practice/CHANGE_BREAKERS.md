# Change Breakers

## Контекст
Эти запахи описывают ситуации, когда изменение в одном месте вызывает каскад
правок в нескольких других частях системы. Контекст важен для оценки стоимости
изменений: чем больше точек модификации, тем выше риск регрессий, конфликтов и
роста времени на поддержку. Такие проблемы редко заметны по функционалу, но
сильно влияют на скорость доставки и надежность. Цель этого блока — заранее
выявлять архитектурные причины «цепной реакции» и устранять их до появления
критических задержек.

## Запахи
### Divergent Change
При внесении изменений в класс приходится изменять большое число различных методов.
Например, для добавления нового вида товара нужно изменить методы поиска, отображения
и заказа товаров.

### Shotgun Surgery
При выполнении любых модификаций приходится вносить множество мелких изменений
в большое число классов.

### Parallel Inheritance Hierarchies
Всякий раз при создании подкласса какого-то класса приходится создавать еще один подкласс
для другого класса.
