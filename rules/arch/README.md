# Arch

## Контекст
Этот каталог содержит архитектурные правила и соглашения. Здесь собраны
описания базовых компонентов доменной модели и практики рефакторинга, которые
используются как единый источник истины для команды.

## Содержимое
- `components/` — правила по ключевым архитектурным компонентам (агрегаты,
  команды, события, запросы, домен).
- `db/` — политики проектирования и эксплуатации базы данных.
- `environment/` — правила эксплуатации и окружения приложения.
- `functions/` — практики и техники рефакторинга функций и логики.
- `ARCH_TESTS.md` — автоматическая валидация DDD-контрактов (R1–R5).
- `LOGS.md` — правила логгирования и контракт формата логов.
- `MONITORING.md` — правила мониторинга и метрик.
- `SERVICES.md` — правила сервисного слоя.
- `VIEWS.md` — правила презентационного слоя.
