# LLM_SECURITY.md — правила безопасности LLM-интеграций

# Контекст
Документ фиксирует правила обращения с выводом LLM. LLM — это недетерминированный внешний сервис. Его ответы обрабатываются как недоверенный input.

## Главное правило
- LLM-output НИКОГДА не используется для принятия решений о доступе, авторизации, аутентификации.
- Решения о доступе — только детерминированный код с явной логикой.
- Нет исключений. Даже "временно". Даже "в dev".

## LLM-output = недоверенный input
- Вывод LLM обрабатывается как данные от внешнего пользователя.
- Валидация обязательна: структура, типы, диапазоны значений.
- Невалидный ответ → явная ошибка, а не fallback на "значение по умолчанию".
- LLM-output не передаётся в SQL, shell, eval, exec, template engine без санитизации.
- Логирование LLM-output не содержит PII пользователей (prompt может содержать PII).

## Prompt Injection Prevention
- Пользовательский ввод изолируется от системных инструкций.
- Разделение: system prompt (контролируемый) vs user content (недоверенный).
- Пользовательский текст не конкатенируется в system prompt напрямую.
- Если LLM парсит внешний контент (HTML, PDF) — результат считается потенциально отравленным.

## Границы ответственности LLM
- LLM формирует контент: текст, оценки, рекомендации, вопросы.
- LLM НЕ принимает решения: маршрутизация, доступы, выполнение команд, управление состоянием.
- Любой score/rating от LLM — информационный, не управляющий.
- Если бизнес-логика зависит от LLM-оценки — пороги и решения определяются детерминированным кодом.

## Structured Output
- Ответ LLM парсится в строго типизированную структуру (dataclass, Pydantic model).
- Парсинг может упасть — обработка ошибки обязательна.
- Fallback-значения допустимы только для необязательных полей, не для критичных.
- Retry-логика ограничена (max retries) и логируется.

## Антипаттерны (запрещено)
- `if llm_response.role == "admin"` — авторизация через LLM
- `eval(llm_response.code)` — выполнение LLM-output как кода
- `f"You are {llm_response.persona}"` — LLM-output в system prompt
- Передача raw LLM-output в SQL-запросы или shell-команды
- Доверие LLM-output без валидации структуры и типов
- Использование LLM для генерации URL, путей к файлам или идентификаторов для прямого использования
