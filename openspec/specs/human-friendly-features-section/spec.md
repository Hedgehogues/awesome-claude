## Requirements
### Requirement: Capability отображается через title, а не через name

Если в `.sdd.yaml` capability записан как объект с полем `title`, секция `## Реализованные фичи` SHALL использовать `title` как отображаемое имя.

#### Scenario: Title подставляется вместо kebab-name

- **GIVEN** `.sdd.yaml` содержит `creates: [{name: sdd-state-skill-hooks, title: "Хуки состояния для скиллов"}]`
- **WHEN** `sdd:apply` рендерит `## Реализованные фичи`
- **THEN** строка содержит «Хуки состояния для скиллов»
- **THEN** строка НЕ содержит «sdd-state-skill-hooks»

#### Scenario: Fallback на name если title отсутствует

- **GIVEN** `.sdd.yaml` содержит `creates: [some-capability]` (строка без title)
- **WHEN** `sdd:apply` рендерит `## Реализованные фичи`
- **THEN** строка содержит «some-capability»

### Requirement: Статусы отображаются на русском языке

Секция `## Реализованные фичи` SHALL отображать статусы `done` и `partial` на русском языке.

#### Scenario: done рендерится как «готово»

- **WHEN** capability имеет статус `done`
- **THEN** строка содержит «готово»
- **THEN** строка НЕ содержит bare «done»

#### Scenario: partial рендерится как «частично»

- **WHEN** capability имеет статус `partial`
- **THEN** строка содержит «частично»
- **THEN** строка НЕ содержит bare «partial»

### Requirement: Причина partial выражена счётчиком задач, не сырым текстом

Для partial-capability причина SHALL быть в форме «(N задач не завершено)», а не дословными текстами незакрытых задач из `tasks.md`.

#### Scenario: Partial показывает счётчик незавершённых задач

- **GIVEN** capability имеет статус `partial` и 2 незакрытые задачи
- **WHEN** `sdd:apply` рендерит строку capability
- **THEN** строка содержит «задач не завершено»
- **THEN** строка НЕ содержит тексты типа «integration тесты требуют live-run» или другой raw task-текст

### Requirement: «Как проверить» использует title capability, не kebab-name

Каждый пункт секции `## Как проверить` SHALL использовать `title` capability в заголовке (если задан).

#### Scenario: Заголовок пункта содержит title, а не kebab-name

- **GIVEN** `.sdd.yaml` содержит `creates: [{name: sdd-state-skill-hooks, title: "Хуки состояния для скиллов"}]`
- **WHEN** `sdd:apply` рендерит `## Как проверить`
- **THEN** заголовок пункта содержит «Хуки состояния для скиллов»
- **THEN** заголовок пункта НЕ содержит «sdd-state-skill-hooks»

### Requirement: Поле «Что» в «Как проверить» описывает ожидаемое поведение, а не реализацию

Поле «Что» в каждом пункте `## Как проверить` SHALL описывать что пользователь ожидает увидеть / почувствовать, а не детали реализации (имена файлов, технические инварианты кода).

#### Scenario: «Что» формулируется от пользователя, не от кода

- **WHEN** `sdd:apply` рендерит пункт проверки для capability «Хуки состояния для скиллов»
- **THEN** поле «Что» звучит как «Скиллы автоматически обновляют состояние change без ручных команд»
- **THEN** поле «Что» НЕ содержит «skill.md файлы не содержат», «transition-вызовов», имён внутренних скриптов

### Requirement: Поле «Как» в «Как проверить» начинается с понятного ожидания

Поле «Как» SHALL начинаться с описания ожидаемого результата на русском языке. Команда для проверки, если нужна, идёт после описания как уточнение, а не вместо него.

#### Scenario: «Как» содержит ожидание перед командой

- **WHEN** `sdd:apply` рендерит поле «Как» для пункта проверки
- **THEN** поле начинается с предложения на русском языке («После запуска apply state-файл содержит stage: proposed»)
- **THEN** если нужна команда, она идёт после описания, а не вместо него

#### Scenario: «Как» не начинается с raw shell-команды

- **WHEN** поле «Как» содержит shell-команду
- **THEN** перед командой есть хотя бы одно предложение с ожиданием
- **THEN** поле «Как» НЕ начинается с `grep`, `cat`, `curl` или другой команды без контекста

### Requirement: Каждый пункт «Что делать» содержит подсказку для уточнения

Каждая строка `- [ ]` в секции `## Что делать` SHALL заканчиваться подсказкой в скобках, которая говорит пользователю как взаимодействовать с этим пунктом: попросить упростить если непонятно, или детализировать если слишком очевидно.

#### Scenario: Подсказка добавлена к каждому пункту

- **WHEN** `sdd:contradiction` рендерит пункт в `## Что делать`
- **THEN** строка заканчивается на `(непонятно — скажи «поясни», слишком очевидно — скажи «детали»)`
- **THEN** подсказка одинакова для всех пунктов

#### Scenario: Пункт без подсказки недопустим

- **WHEN** в `## Что делать` есть строка без скобочной подсказки
- **THEN** это нарушение формата

### Requirement: «Вопросы к пользователю» содержат подсказку для уточнения

Каждый вопрос в блоке `## Вопросы к пользователю` SHALL заканчиваться подсказкой `(непонятно — скажи «поясни», слишком очевидно — скажи «детали»)` — по той же схеме, что и пункты в `## Что делать`.

#### Scenario: Подсказка добавлена к вопросу

- **WHEN** `sdd:contradiction` рендерит вопрос в `## Вопросы к пользователю`
- **THEN** строка заканчивается на `(непонятно — скажи «поясни», слишком очевидно — скажи «детали»)`

### Requirement: Ответ на «поясни» — максимум одно предложение

Если пользователь отвечает «поясни», «не понял», «сложно» или аналогично на любой блок вывода скилла, следующий ответ SHALL быть не длиннее одного предложения.

#### Scenario: Пользователь говорит «не понял»

- **GIVEN** скилл вывел вопрос или пункт «Что делать»
- **WHEN** пользователь отвечает «не понял» или «поясни»
- **THEN** следующий ответ — ровно одно предложение на русском языке
- **THEN** в этом предложении нет технических терминов, имён файлов, вариантов реализации

#### Scenario: Повторный «поясни» не добавляет подробностей

- **GIVEN** скилл уже ответил одним предложением на «поясни»
- **WHEN** пользователь снова говорит «не понял»
- **THEN** скилл переформулирует иначе, но снова одним предложением — без расширения объёма

### Requirement: sdd:propose предлагает добавить title при создании string-capability

Если пользователь добавил capability в `creates` как строку, `sdd:propose` SHALL предложить добавить `title` с объяснением зачем это нужно. Сам по себе `title` не обязателен — его отсутствие не блокирует выполнение.

#### Scenario: Подсказка показывается после записи string-capability

- **GIVEN** `sdd:propose` записал в `.sdd.yaml` `creates: [some-capability]`
- **WHEN** шаг завершён
- **THEN** скилл выводит предложение заменить строку на объект `{name: some-capability, title: "..."}`
- **THEN** если пользователь отказывается — скилл продолжает без блокировки

#### Scenario: Подсказка не показывается если title уже задан

- **GIVEN** `sdd:propose` записал `creates: [{name: some-capability, title: "Понятное название"}]`
- **WHEN** шаг завершён
- **THEN** подсказка про title не выводится

### Requirement: «Вопросы к пользователю» содержат только вопросы про желаемое поведение, не про реализацию

Блок `## Вопросы к пользователю` SHALL содержать только вопросы, на которые можно ответить не зная кода. Технические выборы (как именно реализовать, какую структуру данных использовать, какой флаг передать) Claude закрывает самостоятельно и фиксирует в `## Решено самостоятельно`.

#### Scenario: Вопрос сформулирован через последствия, а не реализацию

- **WHEN** в скилле есть развилка с несколькими вариантами реализации
- **THEN** если разница между вариантами видна только в коде — Claude выбирает сам, вопрос не задаётся
- **THEN** если разница видна в поведении для пользователя — вопрос формулируется через это поведение

#### Scenario: Вопрос можно задать человеку без технического контекста

- **GIVEN** вопрос про удаление файла состояния после архивирования
- **THEN** допустимая формулировка: «После архивирования change'а файл состояния удаляется автоматически. Нужно ли его сохранять?»
- **THEN** недопустимая формулировка: «Как передать флаг `final: true` хуку? Варианты: (а) `state_manager.py` пишет `final: true` в `.sdd-state.yaml`; (б) добавить `__final` в `pending_transitions`»

#### Scenario: Технический выбор уходит в «Решено самостоятельно»

- **WHEN** Claude выбирает между несколькими технически эквивалентными способами реализации
- **THEN** выбор и краткое обоснование появляются в `## Решено самостоятельно`
- **THEN** в `## Вопросы к пользователю` этот выбор не упоминается
