## Context

Скилл `skill:test-skill` записывает результаты прогонов в `test-results/<timestamp>.md`. Директория трекается git'ом через `.gitkeep`. Соглашение в репозитории — скрытые директории для runtime-артефактов (`.sdd-state.yaml`, `.openspec.yaml`). Нужно привести путь вывода в соответствие с этим соглашением.

## Goals / Non-Goals

**Goals:**
- Переименовать `test-results/` → `.test-results/` и добавить в `.gitignore`
- Обновить все строковые упоминания пути в коде, спеках и change-документах
- Удалить `.gitkeep` (после добавления в `.gitignore` он не нужен)

**Non-Goals:**
- Изменение формата файлов результатов
- Изменение логики самого тест-прогона
- Обновление CHANGELOG или версий

## Decisions

**Скрытая директория вместо обычной.**
Имена вида `.foo/` — устоявшееся соглашение для инструментальных директорий (`.git/`, `.pytest_cache/`, `.venv/`). Это даёт мгновенную читаемость: результаты — не часть проекта, а побочный продукт инструмента. Альтернатива — `tmp/test-results/` или `out/` — менее выразительна и требует настройки `.gitignore` точно так же.

**Удалить `.gitkeep`, не заменять.**
После добавления `.test-results` в `.gitignore` сохранять placeholder не нужно: директория создаётся скиллом при первом запуске (`mkdir -p`).

**Обновить pending changes (install-modes, unified-test-flow).**
Эти change'ы ещё не заархивированы, их spec-черновики содержат `test-results/`. Обновление на месте безопасно — они ещё не попали в `openspec/specs/`.

## Risks / Trade-offs

[Сломанные локальные результаты] → Если в `test-results/` есть чужие файлы (не `.gitkeep`), они потеряются при rename. Митигация: `git mv` сохраняет историю; `.gitkeep` — единственный файл, убедиться через `ls`.

[Пропущенные упоминания] → grep по `test-results` перед закрытием задачи.
