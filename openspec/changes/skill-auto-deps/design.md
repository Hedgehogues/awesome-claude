## Context

Скиллы awesome-claude вызывают друг друга через Skill tool. Если вызываемый скилл не установлен в `.claude/skills/`, вызов падает. Автор скилла сейчас не может объявить эту зависимость — пользователь узнаёт о проблеме только при ошибке.

## Goals / Non-Goals

**Goals:**
- Поле `requires:` в frontmatter декларирует зависимости от других скиллов
- Claude проверяет наличие зависимостей при вызове и устанавливает недостающие через скилл `<namespace>:bump-version`
- Синтаксис покрывает скиллы из любого namespace (dev, sdd, openspec и др.)

**Non-Goals:**
- Не покрывать зависимости от внешних CLI-инструментов (отдельная задача)
- Не поддерживать семантическое версионирование зависимостей (semver, диапазоны)
- Не менять поведение Claude Code на уровне хуков

## Decisions

**Синтаксис `requires:` только с типом `skill:`**

```yaml
requires:
  - skill: dev:tracing
    namespace: dev
    repo: git@github.com:Hedgehogues/awesome-claude.git
  - skill: dev:tdd
    namespace: dev
    repo: git@github.com:Hedgehogues/awesome-claude.git
    ref: a1b2c3d
  - skill: openspec-propose
    namespace: openspec
    repo: git@github.com:Hedgehogues/awesome-claude.git
```

`skill:` — имя скилла. `namespace:` — namespace для установки. `repo:` — ссылка на репозиторий-источник. `ref:` — опциональный коммит; если не указан, используется main/master (или аналог дефолтной ветки репозитория).

Проверка: файл `.claude/skills/<namespace>/<skill-name>.md` существует.
Установка: скилл `<namespace>:bump-version` через Skill tool.

**Установка реализуется как преамбула в теле скилла**

Claude Code не имеет хука "до запуска скилла". `requires:` — инструкция для Claude: перед основной логикой проверь зависимости и разрешай недостающие. Преамбула добавляется в тело скилла явно.

**Разработчик декларирует, Claude исполняет**

Автор скилла только перечисляет `requires:`. Логика проверки/установки — стандартная преамбула, одинаковая для всех скиллов.

## Risks / Trade-offs

[Claude может пропустить проверку] → промпт-инструкция не гарантирована. Митигация: покрывается тест-спекой в `__dev`.

[`<namespace>:bump-version` недоступен] → если namespace не установлен. Митигация: при отсутствии скилла Claude сообщает команду для ручной установки: `bash scripts/install.sh <namespace>`.

[Циклические зависимости] → скилл A зависит от B, B зависит от A. На практике не вызывает бесконечного цикла: пользователь вызвал A — файл A уже есть, при установке B Claude проверяет зависимость A, находит файл, продолжает. Цикл разрешается сам. Документируется в SKILL_DESIGN.md как антипаттерн во избежание путаницы.
