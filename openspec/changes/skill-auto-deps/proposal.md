## Why

Скиллы зависят от других скиллов: `dev:fix-bug` вызывает `dev:tracing` + `dev:tdd`, `sdd:propose` вызывает `openspec-propose`. Если нужный скилл не установлен, вызов падает без объяснений. Нет стандартного способа объявить зависимость скилла от другого скилла и автоматически разрешить её при первом вызове.

## What Changes

- Добавить в frontmatter скиллов поле `requires:` — список зависимых скиллов с указанием namespace
- При вызове скилла Claude проверяет наличие каждой зависимости: если файл `.claude/skills/<namespace>/<skill>.md` отсутствует, Claude клонирует недостающие зависимости в `/tmp` и изучает их структуру — ищет любой способ установки скилла: установщик, скрипты, документацию. Если явного механизма нет — придумывает способ самостоятельно исходя из структуры файлов
- Добавить `requires:` во все скиллы с межскилловыми зависимостями — текущие и будущие

## Impact

- `skills/sdd/propose.md` — `skill: openspec-propose`
