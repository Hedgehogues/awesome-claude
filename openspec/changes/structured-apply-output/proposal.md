## Why

Сейчас вывод Claude после `sdd:apply` (и в целом — обращённые к пользователю сообщения) — сплошной AI-текст: технические факты, прозаическое описание, вопросы к пользователю и сторонние замечания смешаны в одном потоке. Пользователь не может за один взгляд понять «что точно сделано», «о чём со мной говорят» и «на какие вопросы я обязан ответить». При этом по итогам `sdd:apply` нет явного списка реализованных фич — нужно реконструировать его из прозы.

См. `.sdd.yaml` для capability-декларации.

## What Changes

- Ввести обязательную **трёхчастную структуру** вывода скиллов, обращённого к пользователю:
  1. **Технические статусы** — факты: пути файлов, результаты тестов, exit codes, состояние артефактов. Без прозы.
  2. **Описание** — короткая прозаическая сводка того, что произошло.
  3. **Вопросы к пользователю** — минимальное число кратких, ёмких, простых вопросов, **на которые пользователь обязан ответить**. Один вопрос — одна строка.
- «Прочие вопросы» (риторические, размышления вслух, exploratory) отделяются от блока 3 и идут отдельным блоком (или не выводятся пользователю).
- В `sdd:apply` после реализации задач: **явный финальный блок «Реализованные фичи»** — нумерованный список фич уровня capability с указанием статуса (done/partial) и ссылкой на spec.
