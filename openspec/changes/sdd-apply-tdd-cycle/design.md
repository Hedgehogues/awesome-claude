## Context

`sdd:apply` делегирует реализацию `openspec-apply-change` CLI, который итерирует по `tasks.md` и выполняет задачи. Артефактная верификация (L1/L2/L3) встроена через `merge-verify-into-apply-archive` — но это пост-фактум проверка наличия и качества артефактов.

Текущий gap: нет enforcement'а, что тесты запускались до реализации (RED) и после (GREEN). Исполнитель мог написать тест после, пропустить RED-фазу или вовсе не запустить тесты.

`run_test.py` (`skill-test-runner-script`) — детерминированный скрипт, который запускает `cases/<ns>/<skill>.md` через `claude -p` и возвращает pass/fail. Он уже умеет SKIP при отсутствии cases-файла.

## Goals / Non-Goals

**Goals:**
- Встроить вызов `run_test.py` в task-loop `sdd:apply`: до задачи → RED-check, после → GREEN-check
- SKIP при отсутствии cases-файла — не блокировать скиллы без тестов
- Предупреждение если до задачи уже GREEN (нет красного теста)
- Hard stop если после задачи не GREEN

**Non-Goals:**
- Не менять формат cases-файлов или стабов
- Не добавлять новую команду пользователю (встраивается в существующий apply)
- Не требовать 100% coverage — SKIP всегда ок для скиллов без кейсов

## Decisions

### D1: Встраивание в task-loop, не отдельный шаг

TDD-цикл встраивается внутрь loop'а каждой задачи `openspec-apply-change`, не добавляется как отдельный финальный шаг. Причина: RED-check должен быть именно до реализации конкретной задачи, не до всего apply.

Альтернатива: запустить тесты один раз в начале apply и один раз в конце → отвергнута: не даёт гарантию red→green per-task.

### D2: GREEN-after = hard stop, RED-before = warning

Если после реализации задачи тесты не GREEN — полная остановка (как break-stop.md). Если до задачи тесты уже GREEN — предупреждение, не остановка: возможны легитимные случаи (первая задача в цепочке, stubs-only изменение).

### D3: SKIP при отсутствии cases-файла

`run_test.py` возвращает exit 0 + сообщение `SKIP: no test spec for <ns>:<skill>`. `sdd:apply` интерпретирует SKIP как "нет тестов — не блокируем". Это позволяет применять TDD-цикл постепенно — только к скиллам с кейсами.

### D4: Prerequisite по capability name

Этот change требует двух capabilities в `openspec/specs/index.yaml`:
- `sdd-apply` (из `merge-verify-into-apply-archive`) — apply workflow должен быть обновлён
- `skill-test-runner-script` (наш `run_test.py`) — детерминированный тест-runner

Зависимость фиксируется в `.sdd.yaml` → `requires:` (информационно, без machine-check).

## Risks / Trade-offs

- `run_test.py` вызывает `claude -p` — каждый тест-прогон стоит токены и время. Митигация: SKIP ускоряет прогон для скиллов без кейсов; на начальном этапе кейсов немного.
- RED-check до задачи может дать ложный SKIP (кейсы есть, но все GREEN) → задача реализована без RED. Митигация: предупреждение в output явно указывает на это.
- Если `run_test.py` недоступен — apply падает с ошибкой. Митигация: проверка наличия скрипта в начале apply; при отсутствии — предупреждение, не hard stop.
