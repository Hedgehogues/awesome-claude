## Context

`skill:test-skill` материализует стабы из `skills/skill/test-skill/stubs/`. Все скиллы используют одни и те же стабы — нет способа адаптировать окружение под конкретный скилл без влияния на остальных. Стаб не принадлежит скиллу, который его использует.

## Goals / Non-Goals

**Goals:**
- Стабы коlocated со скиллом — полная автономность тестового окружения
- Глобальный пул как starter kit: шаблоны для новых скиллов, не shared source of truth
- Fallback-резолюция: local → global, без изменения формата кейсов
- Зафиксировать что расхождение версий стабов намеренно

**Non-Goals:**
- Синхронизация стабов между скиллами
- Валидация идентичности локальных и глобальных копий
- Изменение формата самих стабов (YAML frontmatter остаётся)

## Decisions

### D1: Local-first резолюция

`run_test.py` и `skill:test-skill` ищут стаб в порядке:
1. `skills/<ns>/<skill>/stubs/<name>.md`
2. `skills/skill/test-skill/stubs/<name>.md` (fallback)

Если не найден в обоих — ошибка с указанием обоих путей.

### D2: Глобальный пул — starter kit, не canonical source

`skills/skill/test-skill/stubs/` не является source of truth. Это шаблоны. Создаёшь новый скилл — копируешь нужный стаб, адаптируешь. Изменение в глобальном пуле не требует обновления локальных копий. Изменение в локальной копии не требует обновления глобального пула.

### D3: Расхождение версий — ожидаемое поведение

Локальный `fresh-repo` у `dev:tracing` может отличаться от глобального `fresh-repo`. Это нормально — у каждого скилла своё окружение. Никакой синхронизации не нужно, никакого предупреждения не выводится.

### D4: Существующие кейсы не меняются

Формат `stub: fresh-repo` в `cases/*.md` остаётся. Меняется только где `run_test.py` ищет файл. Это обратно совместимо: все существующие кейсы продолжают работать через fallback на глобальный пул.

## Risks / Trade-offs

- Разработчик не знает что стаб нашёлся через fallback → `run_test.py` логирует: `stub resolved: global fallback (skills/skill/test-skill/stubs/fresh-repo.md)`. Митигация: явный вывод пути.
- Глобальный пул может устареть как starter kit → документировать в README стабов что это шаблоны, не обновлять автоматически.
