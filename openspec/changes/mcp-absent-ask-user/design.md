## Context

`mcp-script-access.md` описывает поведение Claude при отсутствии MCP-интерфейса: остановиться, предложить реализацию, ждать согласия. Формат взаимодействия не задан — правило допускает свободный текст.

AskUserQuestion — инструмент Claude Code, который отображает структурированный вопрос с фиксированным набором вариантов. Пользователь выбирает из списка, а не читает объяснение.

## Goals / Non-Goals

**Goals:**
- Зафиксировать в правиле, что при отсутствии MCP Claude вызывает AskUserQuestion
- Определить два варианта ответа и поведение Claude для каждого

**Non-Goals:**
- Изменять scope правила или условия его срабатывания
- Определять формат описания MCP-тула (это остаётся свободным текстом после выбора)

## Decisions

**D1: Два варианта, не три.**

«Реализовать MCP-интерфейс» и «Продолжить без MCP» — полное покрытие возможных намерений. Третий вариант («Отменить») избыточен: пользователь может просто не отвечать.

**D2: После выбора «Реализовать» — свободный текст с описанием тула.**

AskUserQuestion фиксирует намерение; детали реализации (имя тула, параметры, шаблон) Claude описывает следующим сообщением. Пытаться уместить всё в вопрос — нечитаемо.

## Risks / Trade-offs

**R1: AskUserQuestion может не поддерживаться в некоторых средах.**

В средах без поддержки инструмента Claude вернётся к свободному тексту. Митигация: правило описывает желаемое поведение; деградация до текста лучше, чем молчание.
