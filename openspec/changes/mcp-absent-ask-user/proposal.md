## Why

Правило `mcp-script-access.md` предписывает Claude остановиться и предложить реализацию MCP-интерфейса, если он отсутствует. Однако формат этого предложения не задан: Claude выводит свободный текст, который легко пропустить среди остального вывода.

AskUserQuestion выдаёт структурированный вопрос с фиксированными вариантами ответа — пользователь видит чёткий выбор, а не читает объяснение в потоке текста.

## What Changes

- Обновить секцию «Если MCP-интерфейс отсутствует» в `.claude/rules/mcp-script-access.md`: вместо свободного текста Claude вызывает AskUserQuestion с двумя вариантами — «Реализовать MCP-интерфейс» и «Продолжить без MCP»
- При выборе «Реализовать» — Claude описывает нужный тул и предлагает шаблон
- При выборе «Продолжить» — Claude продолжает с пометкой `⚠ прямой вызов без MCP-интерфейса`

## Capabilities

### New Capabilities

- `mcp-absent-ask-user`: при отсутствии MCP-интерфейса Claude задаёт структурированный вопрос через AskUserQuestion вместо свободного текста

## Impact

**`.claude/rules/mcp-script-access.md`** — секция «Если MCP-интерфейс отсутствует»: заменить описание поведения на вызов AskUserQuestion с двумя вариантами.

См. `.sdd.yaml` для capability-деклараций.
