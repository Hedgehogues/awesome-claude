## Context

`identity.py` — единственное место резолюции email-владельца в SDD-workflow. Сейчас `resolve()` пробует `from_claude()`, при неудаче — `from_git()`. Git-конфиг глобальный и содержит email того, кто первым настроил машину (типично — автор репозитория). Пользователь не подозревает о подмене: скрипт молча возвращает чужой email.

## Goals / Non-Goals

**Goals:**
- Убрать `from_git()` из пути резолюции: источник один — `claude auth status --json`
- При отсутствии Claude-email скрипт завершается с exit ≠ 0 и понятным сообщением

**Non-Goals:**
- Не менять логику `from_git()` как функции (можно оставить как утилиту, если нужна в других контекстах)
- Не менять формат вывода при успешном резолве
- Не добавлять другие источники (env-переменные, config-файл и т.д.)

## Decisions

**Убрать git fallback полностью, не депрецировать.**
Частичное депрецирование (warnings) оставляет дыру: пользователь может не читать stderr. Полное удаление из `resolve()` — единственный способ исключить silent мисатрибуцию. Если `from_git()` нужна кому-то явно, она остаётся как функция, но в `resolve()` не вызывается.

**Сообщение об ошибке — только `claude auth login`.**
Не предлагаем `git config` как альтернативу: это именно то, что создало проблему. Одно действие — одна инструкция.

## Risks / Trade-offs

[Регрессия у пользователей без активной Claude-сессии] → Теперь они получат ошибку вместо молчаливого дефолта на git. Это желательное поведение — явная ошибка лучше скрытой мисатрибуции.

[Сломает тесты, которые мокают git, но не claude] → При архиве нужно убедиться, что case-файлы для `identity.py` обновлены под новое поведение.
