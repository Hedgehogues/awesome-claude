## Why

Пользователи не всегда знают, какой скилл подходит для их задачи. Когда Claude не рекомендует конкретную команду, пользователь теряет ценность от установленного awesome-claude: он решает задачу в режиме «свободного диалога» вместо использования готовых рабочих процессов.

Правило, обязывающее Claude указывать подходящий `namespace:skill` всякий раз, когда это возможно, — самый прямой способ повысить дискаверабилити и утилизацию скиллов.

## What Changes

- Новый файл `.claude/rules/recommend-skills.md` — глобальное правило (без `paths:` frontmatter): при любой задаче Claude должен проверять, покрывает ли её один из доступных публичных скиллов, и если да — назвать конкретную команду до начала выполнения.
- Правило читает список публичных неймспейсов из `manifest.yaml` (секция `namespaces:`); `manifest.yaml` — единственный SSOT для этого списка.
- Правило явно запрещает упоминать внутренние/экспериментальные неймспейсы (например, `opsx:`): они являются зависимостями инфраструктуры, а не пользовательским интерфейсом.
- Правило обязывает Claude при обнаружении устаревшего релиза неймспейса (через `manifest.yaml` или явную просьбу пользователя) предлагать запустить соответствующий `namespace:bump-version` скилл.
