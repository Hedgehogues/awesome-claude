## 3. Правило claude-way

- [x] 3.1 Создать `rules/claude-way.md` со следующими тремя разделами:
  - **Единственный интерфейс**: пользователь взаимодействует только через Claude Code; прямые системные вызовы (bash, curl) не являются частью публичного API
  - **Гранулярность скилла**: последовательность шагов — это bounded context; скилл = 1 шаг; шагов мало (1–2, не более 3); артефакт живёт внутри своего шага — следующий шаг принимает выходы одного или нескольких предыдущих шагов как свои входные артефакты; каноническая форма — 2 этапа по аналогии с ETL: сырой артефакт → преобразованный артефакт
  - **Жизненный цикл артефакта**: каждый создаваемый артефакт декларирует область ответственности (что входит, что нет) и время жизни (ephemeral / session / persistent)
  - **Структурированные данные**: любой доступ (чтение и запись) к YAML, JSON, TOML, INI, CSV, БД, `.versions`, `.manifest` выполняется через Python-скрипт; lifetime скрипта = lifetime трансформации (ephemeral / persistent)
