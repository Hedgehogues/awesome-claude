## Why

Сейчас `sdd:contradiction`, `sdd:apply`, `sdd:archive` требуют передачи имени ченджа через `$ARGUMENTS`. Если аргумент пропущен, скилл либо останавливается с ошибкой, либо задаёт неструктурированный вопрос. Пользователь вынужден помнить точное имя ченджа при каждом вызове.

При этом контекст для определения ченджа уже доступен: ветка git, список недавно изменённых файлов, untracked-директории в `openspec/changes/`. Claude может определить наиболее вероятный чендж сам — но должен подтвердить выбор через список, а не действовать молча.

Отдельная проблема: `sdd:contradiction` правит не только файлы ченджа, но иногда затрагивает рулы, скиллы и другие файлы — выходя за границы bounded context ченджа.

См. `.sdd.yaml` для capability declarations.

## What Changes

- В `skills/sdd/contradiction/skill.md`, `skills/sdd/apply/skill.md`, `skills/sdd/archive/skill.md`: добавить шаг **Determine Change** с двумя режимами:
  - **Контекст очевиден** (например, скилл вызван сразу после другого sdd-скилла и чендж понятен из разговора) — не сканировать файлы, не вызывать инструменты. Показать нумерованный список вариантов прямо в чате через `AskUserQuestion`: «1. Продолжить с `<name>`, 2. Выбрать другой чендж».
  - **Контекст неоднозначен** — показать список всех активных ченджей из `openspec/changes/` через `AskUserQuestion` и дождаться выбора. Ранжирование не нужно — пользователь сам знает, что ему нужно.
- В `skills/sdd/contradiction/skill.md`: добавить явное ограничение — все правки выполняются исключительно внутри `openspec/changes/<name>/`. Запрещено трогать любые файлы вне директории ченджа (подробности — см. `.sdd.yaml` → design.md D3).

## Capabilities

### New Capabilities

- `sdd-implicit-change-selection`: логика в skill.md — два режима при отсутствии `$ARGUMENTS`. **Режим A** (контекст очевиден): без вызова инструментов, показывает список «продолжить с X / выбрать другой». **Режим B** (контекст неоднозначен): читает список ченджей из `openspec/changes/` и показывает его пользователю для выбора. Применяется в contradiction, apply, archive.

### Modified Capabilities

- `sdd-contradiction`: добавляет шаг `Determine Change` + явное ограничение scope (только файлы выбранного ченджа, MUST NOT трогать файлы вне `openspec/changes/<name>/`).
- `sdd-apply`: добавляет шаг `Determine Change`.
- `sdd-archive`: добавляет шаг `Determine Change`.
