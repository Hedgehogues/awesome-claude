## ADDED Requirements

### Requirement: Каждый запуск SDD-скилла создаёт timestamped лог-файл
При каждом запуске скиллов `sdd:contradiction`, `sdd:apply`, `sdd:propose`, `sdd:archive` внутри change-директории SHALL создаваться файл `.log/<skill>-<YYYYMMDDTHHmmss>.md` с полным техническим отчётом. Директория `.log/` SHALL создаваться автоматически при первом запуске.

#### Scenario: Лог создаётся после запуска contradiction
- **WHEN** пользователь запускает `/sdd:contradiction` на change-директории
- **THEN** в `openspec/changes/<name>/.log/` появляется файл `contradiction-<timestamp>.md`
- **THEN** файл содержит полный вывод детекторов (эквивалент текущего `## Технические статусы`)

#### Scenario: Повторный запуск создаёт новый файл
- **WHEN** пользователь запускает тот же скилл на том же change повторно
- **THEN** создаётся новый файл с новым timestamp
- **THEN** предыдущий лог-файл сохраняется без изменений

#### Scenario: Лог создаётся при отсутствии директории .log/
- **WHEN** `.log/` директория ещё не существует в change-директории
- **THEN** скилл создаёт её и записывает файл без ошибки

### Requirement: .log/ директория не попадает в git
`.log/` директории во всех change-директориях SHALL быть исключены из git через паттерн `**/.log/` в `.gitignore`.

#### Scenario: .log/ не отображается в git status
- **WHEN** скилл записал лог-файл в `.log/`
- **THEN** `git status` не показывает файлы из `.log/` как untracked или modified

### Requirement: Лог contradiction содержит drift-таблицу с весами и комментариями
В лог-файле `contradiction` SHALL присутствовать секция `## Drift` с таблицей: тип issue, количество, вес, итоговые баллы, однострочный комментарий на русском языке о значимости.

#### Scenario: Drift-таблица отображает веса корректно
- **WHEN** contradiction завершает анализ с N hard issues и M warnings
- **THEN** в лог-файле секция `## Drift` содержит строки вида:
  `hard issues | N | ×10 | N×10 | <комментарий>`
  `warnings    | M | ×1  | M    | <комментарий>`
  `drift_score | — | —   | N×10+M | итого`
- **THEN** комментарий описывает значимость простым языком (например: «критично, требует исправления до apply» или «незначительно, можно принять»)
