## Why

Скиллы `sdd:propose`, `sdd:contradiction`, `sdd:apply`, `sdd:archive` выводят технический отчёт прямо в диалог (`## Технические статусы`), засоряя экран деталями, нужными только для аудита. Одновременно пользовательские секции (`## Что делать`, `## Вопросы к пользователю`) написаны в терминах детекторов и структуры файлов — человек, знающий только верхнеуровневую задачу, не может быстро принять решение без погружения в детали.

## What Changes

- Каждый из четырёх скиллов при запуске создаёт timestamped лог-файл в `.log/` директории change'а: `openspec/changes/<name>/.log/<skill>-<timestamp>.md`
- `.log/` добавляется в `.gitignore` — логи не коммитятся
- В лог перемещается весь технический вывод (сейчас: `## Технические статусы`): детекторы, exit-коды, verify-вердикты, сырые JSON-ответы
- В `contradiction`-логе дополнительно: таблица drift-весов с простыми комментариями («почему именно столько очков»)
- Пользовательский вывод в диалоге (## Что делать / ## Вопросы к пользователю) переписывается в человеческий язык: без упоминания файлов, детекторов и структур — только задача и решение
- Ссылка на лог-файл добавляется в конец диалогового вывода одной строкой

## Capabilities

### New Capabilities
- `skill-run-log-archiving`: Каждый запуск скилла пишет полный технический отчёт в `.log/<skill>-<timestamp>.md` внутри change-директории; `.log/` не коммитится
- `human-friendly-skill-output`: Пользовательские секции `## Что делать` и `## Вопросы к пользователю` написаны без технического жаргона и без упоминания структуры файлов; в `contradiction` добавлена таблица drift-весов с комментариями

### Modified Capabilities
- `user-interface-principle`: Диалоговый вывод скиллов становится user-facing-only; технический дамп перемещается в лог-файл

## Impact

- `.claude/skills/sdd/contradiction/skill.md` — изменение `User-facing wrapper`: технический отчёт пишется в файл, `## Что делать` генерируется по-новому
- `.claude/skills/sdd/propose/skill.md` — добавление шага записи лога после шага 11
- `.claude/skills/sdd/apply/skill.md` — добавление шага записи лога перед финальным отчётом
- `.claude/skills/sdd/archive/skill.md` — добавление шага записи лога
- `.gitignore` — добавление `**/.log/` в SDD-секцию
- Зеркальные копии в `skills/sdd/*/skill.md` — обновить синхронно

See `.sdd.yaml` for capability declarations.
