## REMOVED Requirements

### Requirement: sdd:help как выделенный скилл-точка-входа
**Reason**: Claude выполняет git-команды и читает frontmatter скиллов нативно. Явный скилл не добавляет логики и создаёт лишние накладные расходы (поддержка, тест-кейсы).
**Migration**: Используй прямые вопросы Claude («покажи статус workflow», «какие changes активны») — Claude воспроизводит то же поведение без скилла.

#### Scenario: Вызов удалённой команды

- **WHEN** пользователь вызывает `/sdd:help`
- **THEN** Claude отвечает нативно (нет зарегистрированной команды) без потери функциональности

### Requirement: Тест-кейсы для sdd:help
**Reason**: Кейсы тестировали тривиальное форматирование без бизнес-логики.
**Migration**: Кейсы удаляются без замены.

#### Scenario: Файлы кейсов отсутствуют после удаления

- **WHEN** тест-раннер ищет `skills/sdd/help/cases/help.md`
- **THEN** файл не найден; тест-раннер не падает (кейс просто не существует)
