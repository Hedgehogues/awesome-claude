## Context

`sdd:help` — скилл первого шага SDD workflow. Его единственная функция: запустить несколько git-команд, прочитать frontmatter из `.claude/skills/sdd/*/skill.md` и отформатировать сводную таблицу пайплайна. Claude Code выполняет эти операции нативно без специального скилла. Скилл существует исторически — как «точка входа», но на практике не добавляет логики, которую нельзя воспроизвести обычным промптом.

Текущие файлы к удалению:
- `.claude/skills/sdd/help/skill.md` — сам скилл
- `.claude/skills/sdd/help/cases/help.md` — тест-кейсы
- `.claude/commands/sdd/help.md` — точка входа команды
- `skills/sdd/help/` + `commands/sdd/help.md` — зеркальные копии в корне репозитория

## Goals / Non-Goals

**Goals:**
- Удалить все файлы `sdd:help` без сохранения функциональности в другом виде
- Обновить документацию, которая явно ссылается на `/sdd:help`
- Убрать `sdd:help` из нумерованного workflow pipeline в `sdd:propose` skill.md (если там есть хардкод)

**Non-Goals:**
- Создавать замену скиллу (Claude справляется без него)
- Изменять нумерацию `workflow_step` в оставшихся скиллах (она несущественна без `sdd:help`)
- Менять поведение других sdd-скиллов

## Decisions

**D1: Полное удаление без замены**
`sdd:help` не оборачивает сложную логику. Альтернатива — переписать как «thin wrapper» — не имеет смысла, т.к. Claude уже выполняет те же операции нативно при любом запросе пользователя о состоянии workflow.

**D2: Не обновлять `workflow_step` в других скиллах**
Числовые значения `workflow_step` используются только для сортировки внутри `sdd:help`. После его удаления это поле перестаёт иметь значение — трогать нет смысла.

**D3: Документацию обновить точечно**
Только файлы с явным упоминанием `/sdd:help` или «step 1» как entry point — CHANGELOG, README если есть. Массовый рефакторинг не нужен.

## Risks / Trade-offs

[Risk] Пользователи, привыкшие к `/sdd:help` как «отправной точке сессии» → Mitigation: Claude отвечает на «с чего начать» и «покажи статус» нативно; ничего не теряется.

[Risk] Хардкодированный список скиллов в `sdd:help/skill.md` содержит актуальный порядок workflow → Mitigation: порядок закодирован в `workflow_step` frontmatter каждого скилла — он никуда не пропадает.
