## Context

Claude Code загружает правила двумя способами: файлы с `paths:` frontmatter — только при совпадении пути; файлы без frontmatter — всегда. Но «всегда» не означает «гарантированно в активном контексте»: движок может не инжектить правило, если не считает его релевантным текущему диалогу. Проверено на `recommend-skills.md`.

Скиллы (install, audit, bump-version) работают с правилами вслепую — нет структурированного реестра, только `ls .claude/rules/`.

## Goals / Non-Goals

**Goals:**
- Создать `index.md` — единственный файл с двумя путями обнаружения (движок инжектит как global rule + CLAUDE.md называет его по имени), что надёжнее любого одиночного global rule
- Сделать `index.md` machine-readable: YAML-блок с `always:` и `path_scoped:` секциями, которые скиллы парсят программно
- Обновить CLAUDE.md: явная ссылка на index.md как точку входа
- Обновить скиллы install/audit/bump-version: читают index.md

**Non-Goals:**
- Изменение поведения движка загрузки правил
- Замена существующих rule-файлов — они остаются как есть
- Автоматический inject содержимого referenced-файлов

## Decisions

**`index.md` без frontmatter.**
По `meta-rules.md`, единственный способ гарантировать всегда-загрузку — отсутствие frontmatter. Добавление любого YAML frontmatter (даже без `paths:`) потенциально меняет поведение движка. Machine-readable данные размещаются в fenced YAML-блоке внутри тела файла — движок не трогает тело, скрипты парсят его через regex/yaml.

**Тело = always-триггеры + YAML-блок.**
index.md содержит две части: (1) краткий текст для Claude — те инструкции, которые должны срабатывать безусловно; (2) fenced YAML-блок `rules:` — для программного чтения скиллами. Разделение: текст для Claude, YAML для инструментов.

**CLAUDE.md получает секцию `## Rules`.**
Явное упоминание в CLAUDE.md — семантическая гарантия: даже если движок не инжектил index.md, CLAUDE.md (гарантированно в контексте) говорит Claude «точка входа — index.md». Это создаёт второй путь обнаружения.

**Скиллы читают index.md через Bash/python, не через Claude-read.**
Скрипты парсят YAML-блок программно: `grep` + `yaml.safe_load`. Это надёжнее, чем инструктировать Claude читать файл в рамках skill-шага.

## Risks / Trade-offs

[Синхронизация index.md] — При добавлении нового rule-файла нужно обновить index.md вручную. Митигация: в теле index.md явная инструкция «при добавлении правила — обновить этот файл»; sdd:audit проверяет соответствие.

[meta-rules.md конфликт] — `meta-rules.md` говорит «каждый rule-файл MUST иметь `paths:`». `index.md` намеренно нарушает это. Митигация: обновить meta-rules.md, добавив index.md как явное исключение наравне с `STATE_OWNERSHIP.md`.
