## Why

Текущий принцип (`claude-way.md` → «Структурированные данные») требует работать со структурированными данными через Python-скрипты, а не нативно через Bash. Это защищает от ad-hoc операций, но не даёт Claude наблюдаемого, структурированного интерфейса: вызов через `Bash tool` — это прямой доступ с текстовым выводом и позиционными аргументами без схемы.

Та же проблема шире: скрипты — не единственные внешние системы, с которыми работает Claude. Сервисы (HTTP API, базы данных, очереди), CLI-инструменты и любые другие процессы также вызываются через Bash без типизации и контракта.

MCP (Model Context Protocol) предоставляет единый типизированный, версионируемый интерфейс ко всем внешним системам. Если инструмент задействован через MCP-тул, Claude видит JSON-schema аргументов, получает структурированный ответ и не рискует передать неверный путь, формат или порядок аргументов.

Проблема: сегодня нет правила, которое требует наличия MCP-интерфейса до использования любого внешнего инструмента. Claude вызывает скрипты, сервисы и инструменты напрямую, иногда угадывая аргументы по сигнатуре.

## What Changes

- Добавить правило `.claude/rules/mcp-script-access.md`: взаимодействие с внешними системами — через MCP-интерфейс; scope правила открытый и расширяется инкрементально по мере того, как становится ясно, какие системы покрываются
- Если MCP-интерфейс отсутствует — Claude предлагает его реализацию пользователю; при явном согласии пользователя продолжает без MCP с пометкой
- Правило расширяет, не заменяет «Структурированные данные» из `claude-way.md`: скрипт по-прежнему обязателен для структурированных данных, но точка входа теперь — MCP
- Обновить секцию «Структурированные данные» в `claude-way.md`: добавить ссылку на новое правило

## Capabilities

### New Capabilities

- `scripts-access-via-mcp`: Claude взаимодействует со скриптами, сервисами и внешними инструментами строго через MCP-интерфейс; при отсутствии интерфейса — предлагает его реализацию пользователю

## Impact

**`.claude/rules/mcp-script-access.md`** — новый файл правила со scope (скрипты, сервисы, CLI), поведением при отсутствии MCP и форматом предложения реализации.

**`.claude/rules/claude-way.md`** — секция «Структурированные данные»: добавить ссылку `см. mcp-script-access.md`.

См. `.sdd.yaml` для capability-деклараций.
