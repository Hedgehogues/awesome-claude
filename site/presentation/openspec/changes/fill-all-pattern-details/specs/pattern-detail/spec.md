## MODIFIED Requirements

### Requirement: Структура детального контента паттерна
`PatternItem` SHALL поддерживать опциональное поле `detail` со структурой: `problem`, `solution`, `quote`, `worksWhen`, `failsWhen`, `visual`. Все паттерны во всех категориях SHALL иметь заполненный `detail` с минимальным набором полей `problem` и `solution`.

#### Scenario: Отображение структурированного контента
- **WHEN** пользователь открывает паттерн, у которого есть поле `detail` со всеми секциями
- **THEN** overlay отображает: проблема (левая колонка), решение (левая колонка), цитата (левая колонка), инфографика (правая колонка)

#### Scenario: Частичный контент
- **WHEN** пользователь открывает паттерн, у которого `detail` содержит только `problem` и `solution`
- **THEN** overlay отображает только присутствующие секции без пустых блоков

#### Scenario: Graceful fallback для паттерна без детального контента
- **WHEN** пользователь открывает паттерн, у которого нет поля `detail`
- **THEN** overlay отображает крупно title и summary, без секций проблемы/решения/цитаты
