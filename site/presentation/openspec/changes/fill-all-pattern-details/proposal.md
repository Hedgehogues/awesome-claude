## Why

После пилота «Трекер как SSOT» overlay работает, но 72 из 73 паттернов при клике показывают только голый fallback (title + summary крупно). Каталог паттернов — ключевая часть доклада; каждый паттерн должен раскрываться с проблемой, решением, цитатой и визуальной схемой.

## What Changes

- Поле `detail` заполняется для всех паттернов во всех 10 категориях `patterns.ts`
- Источник контента — `research/notes/analysis/engineering-patterns.md` (60+ паттернов с описанием «суть / где встречается / когда работает / когда ломается»)
- Каждый паттерн получает: `problem`, `solution`, `quote` (где есть в источнике), `worksWhen`, `failsWhen`, `visual` (ASCII-схема)
- Паттерны, по которым в источнике нет достаточного материала, получают минимальный `detail` (problem + solution без quote и visual)

## Capabilities

### Modified Capabilities
- `pattern-detail`: расширение — теперь все паттерны имеют `detail`, а не только один пилот

## Impact

- `src/data/patterns.ts` — заполняется поле `detail` для ~72 паттернов (10 категорий)
- Нет изменений в компонентах, CSS, роутинге
